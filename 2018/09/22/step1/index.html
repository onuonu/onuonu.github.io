把自己的html文件放到github里

第一步：新建仓库

![s1-1]/images/step1/s1-1.png)
1.打开settings，有一个Github Pages 的设置，点击 source 中的本来的 None ，使其变成 master 分支，也就是作为部署github pages 的分支，然后点击 save。

2.页面刷新之后，再看 github pages 设置框处，多了一行网址，就是你的 github pages 的网址了。

第二步：创建本地文件

1.打开此电脑，选择一个盘，新建nono文件夹，右键空白处点击 git bash here。

2.克隆test repository到 test 文件中。

3.命令 git clone [https://github.com/okaeli/nono.git]

这个时候你的 e 盘，就会多一个 nono 文件夹，打开它，将自己的网页文件复制粘贴至 f 盘的 nono文件夹中
4.命令git status，列出当前目录所有还没有被git管理的文件和被git管理且被修改但还未提交(git commit)的文件，也就是所有改动文件，红色字体标出。

5.然后输入 git add . (有个点) 表示添加当前目录下的所有文件和子目录。
然后 再输入一次 git status 如果看见文件都变绿了 ，那么就代表 它们已经准备好了被提交（git commit）。

6.输入命令（git commit -m ‘这里是描述test’）将文件上传至远程 master 分支

7.先pull再push
git pull (从远程仓库下载到本地仓库)
git push (从本地仓库上传到远程仓库 )

8.即可访问https://okaeli.github.io/nono/demo4/index.html

第三步：后续添加修改

1.将修改好的文件或添加的文件放入nono文件夹中
2.命令git status
3.然后输入 git add . (有个点) 表示添加当前目录下的所有文件和子目录。
然后 再输入一次 git status 如果看见文件都变绿了 ，那么就代表 它们已经准备好了被提交（git commit）。
4.输入命令（git commit -m ‘这里是描述test’）将文件上传至远程 master 分支
5.先pull再push
